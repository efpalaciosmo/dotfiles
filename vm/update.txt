Neovim 0.12 migration log
=========================

Goal: rewrite the existing Neovim config around the new 0.12 features --
vim.pack, native vim.lsp.config, native autocomplete, built-in treesitter
highlighting, and the redesigned diagnostic config -- while keeping the
adwaita-inspired theme and the lualine bar working.

What was wrong before
---------------------

1.  init.lua required `config.option`, `config.plugins`, `config.theme`,
    `config.keymap`, `config.autocmds`, `config.lsp`, but the actual files
    lived directly under `lua/options.lua`, `lua/plugins.lua`, ... so the
    config was effectively broken until you fixed the paths by hand.
2.  Plugin manager was lazy.nvim. It still works on 0.12 but means an extra
    bootstrap step plus a separate lockfile, which 0.12 already provides.
3.  LSP went through nvim-lspconfig + Mason + mason-lspconfig. With 0.12
    that whole stack can be replaced by `vim.lsp.config[...]` +
    `vim.lsp.enable(...)`.
4.  Diagnostic signs were configured the old way; 0.12 requires going
    through `vim.diagnostic.config({ signs = { text = ... } })`.
5.  Markdown rendering looked broken because:
        - global `conceallevel = 0` killed render-markdown
        - the autocmd that started treesitter ran before checking that a
          parser existed, which can throw on 0.12 where get_parser() now
          returns nil instead of erroring
        - markdown-preview.nvim's install hook was run inside lazy's
          `build` callback, which only fires when lazy is the manager
6.  nvim-treesitter was pinned to its old `master` branch + the deprecated
    `.configs.setup()` API. The plugin was archived; 0.12 ships parsers
    for the basics and the `main` branch is now just a parser installer.

New layout
----------

    ~/.config/nvim/
    |-- init.lua
    |-- ftplugin/
    |   `-- markdown.lua
    `-- lua/
        `-- config/
            |-- options.lua
            |-- diagnostics.lua
            |-- theme.lua
            |-- pack.lua
            |-- lsp.lua
            |-- keymaps.lua
            `-- autocmds.lua

`init.lua` is the only entry point; everything else is `require`d in a
fixed order so the colorscheme is up before plugins paint anything.

Concrete changes
----------------

1. Plugin manager: lazy.nvim -> vim.pack
   - All plugins are declared in `lua/config/pack.lua` with a single
     `vim.pack.add({...})` call (the recommended style from the official
     vim.pack guide).
   - Install hooks are autocmds on `PackChanged`, registered BEFORE the
     `vim.pack.add()` call so they fire on first bootstrap from the
     lockfile too.
   - `nvim-pack-lock.json` will appear next to init.lua; commit it.
   - Removed all lazy.nvim bootstrap code and `lazypath` shim.

2. LSP: nvim-lspconfig + Mason -> native vim.lsp.config
   - `lua/config/lsp.lua` declares each server with `vim.lsp.config[name]
     = { cmd, filetypes, root_markers, settings, ... }` and then
     `vim.lsp.enable({...})` only those whose binary is on $PATH.
   - All Mason / mason-lspconfig / nvim-lspconfig plugins were dropped.
     Install language servers via your distro / volta / pip / cargo.
   - `:checkhealth vim.lsp` now tells you which buffers have which
     clients attached.

3. Diagnostics
   - Migrated to the new `vim.diagnostic.config({ signs = { text = ... },
     virtual_text = ..., float = ... })` shape required by 0.12.
   - `[d` / `]d` use `vim.diagnostic.jump({ count = +-1, float = true })`
     instead of the now-removed `goto_prev` / `goto_next`.

4. Treesitter
   - Removed `nvim-treesitter.configs.setup()` (deprecated, archived).
   - Kept `nvim-treesitter` on the `main` branch purely as a parser
     installer. Setup uses the new `require('nvim-treesitter').install()`
     API and is wrapped in `pcall` so a missing API can't break startup.
   - Highlighting is started by an autocmd on FileType that first checks
     `vim.treesitter.language.get_lang()` and `vim.treesitter.get_parser
     (..., { error = false })`. 0.12 returns nil instead of throwing.
   - Markdown highlighting is on by default in 0.12 and works without any
     config.

5. Completion
   - Removed `blink.cmp`. 0.12 has built-in insert-mode auto-completion.
   - Enabled with `vim.o.autocomplete = true` plus `completeopt =
     menu,menuone,noinsert,noselect,popup,fuzzy`.
   - `pummaxwidth = 40`, `pumheight = 12` keep the popup compact.
   - LSP capabilities are no longer broadcast through blink; the built-in
     client already advertises completion support.

6. Adwaita theme
   - The palette in `lua/config/theme.lua` is unchanged.
   - Added explicit highlights for `@markup.heading.{1..6}.markdown`,
     `@markup.heading.N.marker.markdown`, `@markup.raw.markdown_inline`,
     `@markup.link`, `RenderMarkdownH{1..6}` so markdown looks right.
   - Added `PmenuMatch` / `PmenuMatchSel` so the new native autocomplete
     popup matches the rest of the UI.
   - Re-applies on `OptionSet background` so light/dark toggling works.

7. Lualine bar
   - Same theme palette, re-derived from the adwaita colors.
   - Added a small `lsp_status()` component that uses `vim.lsp.status()`
     (new in 0.12) to show progress messages, falling back to the list of
     attached client names.
   - `globalstatus = true` so we get a single bar at the bottom.

8. Markdown
   - New `ftplugin/markdown.lua` sets `conceallevel = 2`, `wrap`,
     `linebreak`, `spell`, smaller indent, and prose-friendly `j`/`k`
     bindings on wrapped lines, plus a `<leader>x` checkbox toggle.
   - `render-markdown.nvim` is configured with block-style headings,
     bullet glyphs, rounded pipe tables, LSP-backed completions, and
     check-/un-checked icons.
   - `markdown-preview.nvim`'s install step runs from the `PackChanged`
     autocmd so it works under vim.pack.
   - Conform now uses `lsp_format = "fallback"` instead of the deprecated
     `lsp_fallback`.

9. Misc 0.12 polish
   - `vim.loader.enable()` at the top of init.lua (recommended free
     speed-up).
   - `winborder = "rounded"` so every floating window has the rounded
     border, including LSP hovers and code-action menus.
   - `diffopt += "linematch:60"` for nicer side-by-side diffs (0.12 ships
     "indent-heuristic" + "inline:char" defaults already).
   - Autocmd to auto-create missing parent directories on save.
   - Trim trailing whitespace on save (skipping markdown / diff / git
     commit).

How to apply
------------

    # nvim is already 0.12.x in this VM
    cd ~/dotfiles
    stow -t ~ vm/nvim          # or however you symlink configs

    # First launch installs everything via vim.pack:
    nvim
    # Press 'a' on the confirmation dialog to allow all installs.

    # Optional sanity checks:
    :checkhealth vim.pack
    :checkhealth vim.lsp
    :checkhealth vim.treesitter

Install `tree-sitter` (the CLI) if you want extra parsers (astro, go, zig,
typst, ...). Without it the config silently uses 0.12's bundled parsers
(c, lua, vim, vimdoc, query, markdown, markdown_inline, python, bash).

Make sure these binaries are on $PATH for full functionality:
lua-language-server, pyright, typescript-language-server, astro-ls,
vscode-{css,html,json}-language-server, gopls, clangd, zls, marksman,
tinymist, sql-language-server, prettier, stylua, ruff, goimports,
clang-format, markdownlint-cli2, markdown-toc.

Anything missing is silently skipped -- the editor still starts cleanly.

Cleanup of the previous setup
-----------------------------

After confirming the new config works, delete the lazy.nvim leftovers
so they can't shadow vim.pack:

    rm -rf ~/.local/share/nvim/lazy
    rm -rf ~/.local/share/nvim/mason
    rm -f  ~/.config/nvim/lazy-lock.json
